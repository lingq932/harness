# Harness Engineering 通用模板

> 用于约束 AI Agent 在工程项目中的行为，确保可控、可回滚、可交接。

---

## 一、核心理念

Harness 的本质是：**人控盘，AI 执行，全程有据可查。**

- AI 是非确定性的，同样的输入可能产出不同结果
- 因此需要「缰绳」：明确的规格、检查点、回滚机制
- 人是「控盘者」，负责决策和验收；AI 是「执行者」，负责产出和建议

---

## 二、两个核心文件

### 1. spec.md — 单一事实来源

```markdown
# 项目名 — Spec

## 目标
一句话说清楚：做什么、给谁用、核心价值。

## 范围
### 包含
- 功能模块 1：描述
- 功能模块 2：描述
- ...

### 视觉/交互要求
- 风格方向
- 关键约束

## 技术方案
- 框架 / 语言 / 部署方式

## 约束
- 不做什么（明确边界）
- 数据来源限制
- 性能要求

## 暂不处理项
- 明确列出推迟的功能，防止范围蔓延
```

**使用规则：**
- Spec 是唯一的需求来源，AI 不应自行添加 spec 之外的功能
- 任何需求变更必须先更新 spec，再执行
- Spec 由人审批，AI 只能建议修改

### 2. handoff.md — 交接文档

```markdown
# 项目名 — Handoff

## 项目状态
当前阶段 + 线上地址 + 如何运行

## Spec 完成度
表格形式逐项对照

## 文件结构
关键文件说明

## 如何继续迭代
常见操作的具体步骤

## 可继续优化的方向
checklist 形式列出
```

**使用规则：**
- 项目阶段性完成时编写
- 确保任何人（或另一个 AI session）拿到 handoff 就能接手

## 三、8 步工作流 SOP

```
Step 1: 明确目标 → 人描述意图，AI 复述确认
Step 2: 编写 Spec → AI 起草，人审批
Step 3: 任务拆解 → AI 拆成可执行的任务块，人确认顺序
Step 4: Checkpoint → 每个任务块执行前，AI 陈述要做什么、预期产出
Step 5: 执行 → AI 动手，人观察
Step 6: 验收 → 人检查产出，通过则进入下一块
Step 7: 迭代 → 不满意则调整，可回滚到上一版本
Step 8: 交接 → 编写 handoff.md，项目可被任何人接手
```

**关键原则：**
- **一次只做一块**，没有批准不跳到下一块
- **Checkpoint before Execute**：先说再做，不要闷头干
- **人批准才推进**：AI 完成后等待确认，不自动继续

---

## 四、3 层目标管理

| 层级 | 内容 | 谁管 |
|------|------|------|
| 战略层 | 项目目标、核心价值 | 人定义 |
| 战术层 | 任务拆解、优先级排序 | AI 建议，人审批 |
| 执行层 | 具体代码、文件修改 | AI 执行，人验收 |

AI 不应在执行层擅自做战略层的决策（比如自行添加功能）。

---

## 五、4 个漂移信号

当出现以下情况时，立即暂停并与人确认：

| 信号 | 表现 | 应对 |
|------|------|------|
| 范围蔓延 | AI 开始做 spec 之外的事 | 停下来，问人是否要扩展 spec |
| 技术偏离 | 实现方式与 spec 不符 | 回到 spec 确认技术方案 |
| 质量下降 | 产出明显粗糙或有 bug | 先修复再继续，不要带病推进 |
| 沟通断裂 | 人和 AI 理解不一致 | 复述理解，对齐后再继续 |

## 六、版本管理与回滚

```
src/_versions/
├── v1.0/    ← 每次大改前备份
├── v2.0/
└── v3.0/
```

**规则：**
- 每次视觉大改或结构变更前，先备份当前版本
- 命名清晰（v1.0、v2.0），备份目录里放关键文件
- 不满意时可以一键回滚：复制备份文件覆盖当前

---

## 七、角色设置（可选）

在教学或复杂项目中，可以让 AI 扮演双重角色：

- **AI 角色**：执行任务、回答问题、产出代码
- **教练角色**：解释为什么这么做、点评 Harness 方法论的应用

默认对话对象是 AI，除非人明确说"问教练"。

---

## 八、Quick Start Checklist

开始一个新的 Harness 项目时：

- [ ] 创建 `_harness/` 目录
- [ ] 人描述目标，AI 复述确认
- [ ] AI 起草 `spec.md`，人审批
- [ ] AI 拆解任务块，人确认
- [ ] 逐块执行：checkpoint → 执行 → 验收
- [ ] 每次大改前备份版本
- [ ] 完成后编写 `handoff.md`
