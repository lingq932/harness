# 多 Agent 保险续费外呼系统 — 任务拆解示例

> 展示复杂项目如何拆解任务块、定义依赖关系、设置检查点。

## 任务总览（Phase 1 MVP）

```
任务块 1: 数据层搭建          ← 无依赖，最先做
任务块 2: 调度 Agent 开发      ← 依赖任务块 1
任务块 3: 开场 Agent 开发      ← 依赖任务块 1
任务块 4: Agent 间交接联调     ← 依赖任务块 2 + 3
任务块 5: 监控看板             ← 依赖任务块 4
任务块 6: 灰度发布 & 验收      ← 依赖任务块 4 + 5
```

---

## 任务块 1：数据层搭建

**做什么：** 搭建客户数据表、保单数据表、外呼任务队列、对话日志存储

**预期产出：**
- 数据库 schema 文档
- 任务队列读写接口
- 测试数据集（100 条模拟客户 + 保单）

**Checkpoint 要点：**
- 表结构是否覆盖 spec 里 Agent 交接协议所需的全部字段
- 测试数据是否覆盖各种客户类型（新客/续费/高风险/黑名单）

**验收标准：**
- [ ] 数据表创建完成，字段与 spec 一致
- [ ] 任务队列可正常入队/出队
- [ ] 100 条测试数据导入成功

---

## 任务块 2：调度 Agent 开发

**做什么：** 实现客户筛选逻辑、触达时机策略、任务分配

**预期产出：**
- 调度 Agent Prompt v1.0
- 筛选规则配置文件
- 时间策略逻辑

**Checkpoint 要点：**
- 筛选规则是否排除了黑名单客户
- 时间策略是否符合 spec 约束（工作日 10-12/14-17，晚 8 点后禁止）
- 每月 3 次上限是否生效

**依赖：** 任务块 1（需要读取客户和保单数据）

**验收标准：**
- [ ] 100 条测试数据中，正确筛选出应触达客户
- [ ] 黑名单客户未被分配任务
- [ ] 触达时间全部在允许范围内
- [ ] 同一客户月内分配不超过 3 次

---

## 任务块 3：开场 Agent 开发

**做什么：** 实现首轮外呼话术，包括身份确认、来意说明、意向判断

**预期产出：**
- 开场 Agent Prompt v1.0
- 意向标签分类逻辑（有意向/拒绝/未接/犹豫）
- 5 轮模拟对话测试记录

**Checkpoint 要点：**
- Prompt 是否在开场 5 秒内表明 AI 身份（合规）
- 客户说"不要再打"是否触发立即结束
- 意向标签覆盖是否完整

**依赖：** 任务块 1（需要客户画像数据注入 Prompt）

**验收标准：**
- [ ] 合规检查通过（AI 身份声明、拒绝即停）
- [ ] 5 轮模拟对话意向判断准确率 ≥ 80%
- [ ] Prompt 内容经产品 + 合规双重审批

---

## 任务块 4：Agent 间交接联调

**做什么：** 打通调度 → 开场的完整链路，验证交接协议

**预期产出：**
- 端到端测试报告（20 通模拟外呼）
- 交接数据完整性校验结果
- 路由准确率统计

**Checkpoint 要点：**
- 交接 JSON 格式是否与 spec 附录 A 一致
- 开场 Agent 能否正确读取调度 Agent 传来的客户信息
- 意向标签能否正确回传给调度 Agent（用于决定是否重试）

**依赖：** 任务块 2 + 3

**验收标准：**
- [ ] 20 通模拟外呼全部走通完整链路
- [ ] 交接数据无字段缺失
- [ ] 路由准确率 ≥ 90%
- [ ] 对话日志完整记录

---

## 任务块 5：监控看板

**做什么：** 搭建实时数据看板，展示核心指标

**预期产出：**
- 看板页面（接通率、对话完成率、意向分布、错误率）
- 数据刷新频率 ≤ 5 分钟

**Checkpoint 要点：**
- 指标定义是否与 spec 效果评估指标一致
- 是否包含灰度发布对比视图（AI 组 vs 人工组）

**依赖：** 任务块 4（需要真实对话数据）

**验收标准：**
- [ ] 所有技术指标和业务指标均有展示
- [ ] 数据实时刷新正常
- [ ] 灰度对比视图可用

---

## 任务块 6：灰度发布 & 验收

**做什么：** 按 spec 灰度策略逐步放量，监控回退条件

**预期产出：**
- 灰度发布执行记录
- 各阶段指标对比报告
- 最终验收报告

**Checkpoint 要点：**
- 是否从内部测试开始（不直接上真实客户）
- 回退条件是否写入监控告警
- 放量审批是否经过产品 + 运营 + 技术三方确认

**依赖：** 任务块 4 + 5

**验收标准：**
- [ ] 内部测试通过
- [ ] 5% 灰度无回退信号
- [ ] 20% 灰度转化率不低于人工组 80%
- [ ] 产品负责人签字确认可全量

---

## 与入门场景的对比

入门场景（个站）的任务拆解是线性的：任务 1 → 2 → 3 → ... 逐个做。

进阶场景的区别：

| 维度 | 入门（个站） | 进阶（多 Agent） |
|------|-------------|-----------------|
| 任务依赖 | 线性，无依赖 | 有依赖树，部分可并行 |
| 审批角色 | 只有自己 | 产品 + 合规 + 技术 + 运营 |
| 验收标准 | 视觉 + 功能 | 数据指标 + 合规 + 灰度 |
| 回滚粒度 | 整页回滚 | 单个 Agent Prompt 回滚 |
| Checkpoint 重点 | "好不好看" | "会不会出事" |
| 版本管理 | 文件备份 | Prompt 版本 + 灰度策略 |
