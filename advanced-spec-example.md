# 多 Agent 保险续费外呼系统 — Spec（示例）

> 这是一份进阶场景的 spec 示例，用于展示复杂项目中 Harness 规格文档的写法。

## 目标

搭建一套多 Agent 协作的保险续费外呼系统。核心价值：替代人工电话续费，通过多个 AI Agent 分工协作，完成从客户筛选到成单转化的全流程，将触达覆盖率从 35% 提升至 90%+。

## 背景

- 当前人工外呼团队每天触达量有限，大量即将到期保单无人跟进
- 单一 AI 外呼脚本无法应对多样化的客户场景（拒绝、犹豫、价格敏感、需要转人工等）
- 需要多个专职 Agent 各司其职，组合完成一次完整的续费转化

## Agent 架构

### Agent 清单

| Agent 名称 | 职责 | 输入 | 输出 |
|------------|------|------|------|
| 调度 Agent | 筛选客户、分配任务、决定触达时机 | 客户保单数据 | 外呼任务队列 |
| 开场 Agent | 首轮外呼，确认身份，说明来意 | 任务队列 + 客户画像 | 客户意向标签（有意向/拒绝/未接） |
| 异议处理 Agent | 应对价格异议、犹豫、比价等场景 | 客户意向 + 历史对话 | 处理结果（已说服/需升级/放弃） |
| 转化 Agent | 促成最终下单，引导支付流程 | 已说服客户 | 成单/未成单 |
| 转人工 Agent | 识别需要人工介入的场景，整理上下文转交 | 复杂场景对话 | 人工工单 + 摘要 |

### Agent 间交接协议

```
调度 Agent → 开场 Agent
  交接内容：客户 ID、保单信息、最佳触达时间、历史交互记录
  交接格式：JSON，字段见附录 A

开场 Agent → 异议处理 Agent / 转化 Agent / 转人工 Agent
  交接内容：对话记录、客户意向标签、情绪评估
  路由规则：
    - 意向明确 → 转化 Agent
    - 有异议 → 异议处理 Agent
    - 情绪激动/场景复杂 → 转人工 Agent
    - 拒绝/未接 → 回调度 Agent（安排重试或放弃）

异议处理 Agent → 转化 Agent / 转人工 Agent
  交接内容：异议类型、处理记录、客户最新态度
  路由规则：
    - 已说服 → 转化 Agent
    - 3 轮未说服 → 转人工 Agent
```

## 范围

### Phase 1（MVP）
- 调度 Agent + 开场 Agent 两个 Agent 先跑通
- 单一保险产品（车险续费）
- 仅外呼，不含短信/微信触达
- 异议处理和转化暂由人工完成

### Phase 2
- 补齐异议处理 Agent + 转化 Agent
- 增加短信预热触达
- 支持 2-3 个保险产品

### Phase 3
- 转人工 Agent 上线
- 全产品线覆盖
- 灰度发布机制成熟

## 技术方案

- Agent 框架：基于 LLM 的 Prompt 工程，每个 Agent 独立 Prompt
- 话术引擎：Prompt 模板 + 变量注入（客户名、保单信息、价格等）
- 调度系统：规则引擎 + 时间策略（工作日 10-12 / 14-17 点）
- 数据存储：MySQL（客户/保单） + Redis（任务队列） + ES（对话日志）
- 监控：实时看板（接通率、转化率、异议分布）

## Prompt 版本管理

```
prompts/
├── dispatcher/
│   ├── v1.0.md    ← 当前线上版本
│   └── v1.1.md    ← 灰度测试版本
├── opener/
│   ├── v2.0.md
│   └── v2.1.md
├── objection-handler/
│   └── v1.0.md
└── closer/
    └── v1.0.md
```

**版本规则：**
- 每个 Agent 的 Prompt 独立版本管理
- 修改 Prompt 必须新建版本号，不允许覆盖
- 上线前必须经过灰度验证（5% → 20% → 50% → 100%）

## 效果评估指标

### 技术指标
| 指标 | 定义 | 目标 |
|------|------|------|
| 接通率 | 成功接通 / 总外呼 | ≥ 60% |
| 对话完成率 | 完整走完流程 / 接通数 | ≥ 75% |
| Agent 路由准确率 | 正确转交 / 总转交 | ≥ 90% |
| 平均对话时长 | - | 90-180 秒 |

### 业务指标
| 指标 | 定义 | 目标 |
|------|------|------|
| 触达覆盖率 | 已触达客户 / 应触达客户 | ≥ 90% |
| 续费转化率 | 成单数 / 接通数 | ≥ 15% |
| 客户投诉率 | 投诉数 / 外呼数 | ≤ 0.5% |
| 人工升级率 | 转人工数 / 接通数 | ≤ 20% |

### 体验指标
| 指标 | 定义 | 目标 |
|------|------|------|
| 客户满意度 | 通话后评分 | ≥ 4.0/5.0 |
| 话术自然度 | 人工抽检评分 | ≥ 85 分 |

## 灰度发布策略

```
阶段 1：内部测试（团队自己的号码）
阶段 2：5% 真实流量（低风险客户池）
阶段 3：20% 流量（对比人工组效果）
阶段 4：50% 流量（确认无回退信号）
阶段 5：100% 全量上线
```

**回退条件（任一触发立即回退）：**
- 转化率低于人工组 50%
- 投诉率超过 1%
- Agent 路由错误率超过 15%

## 约束

- 通话必须在开场 5 秒内表明 AI 身份（合规要求）
- 客户说"不要再打了"必须立即结束并加入黑名单
- 晚 8 点后不允许外呼
- 每个客户每月最多触达 3 次
- Prompt 修改必须经过产品和合规双重审批

## 多角色审批矩阵

| 变更类型 | 审批人 |
|----------|--------|
| Spec 修改 | 产品负责人 |
| Prompt 内容修改 | 产品 + 合规 |
| Agent 路由规则修改 | 产品 + 技术 |
| 灰度放量 | 产品 + 运营 + 技术 |
| 全量上线 | 产品负责人最终批准 |

## 暂不处理项

- 视频外呼
- 多语言支持
- 客户自助续费入口（App/小程序）
- 与第三方保司系统对接
- AI 声音克隆（使用通用 TTS）
